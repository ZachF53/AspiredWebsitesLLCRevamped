Intake form polish — auto-save, photo uploads, helpful hints, social split, drop GBP-access from intake

Auto-save reliability
  HTMX trigger widened from `change delay:800ms` to
  `change, input changed delay:600ms` so:
    - any tab-out, checkbox click, select change, or file pick fires an
      immediate save (no 800ms delay)
    - typing into text inputs / textareas auto-saves 600ms after the
      last keystroke without needing blur
  Same intake-progress partial swap — no UX surprise, just snappier.

Step 2 — Photos
  - New IntakePhoto model (FK to IntakeResponse, FileField). Upload
    path: portal/intake/photos/<client_id>/<filename>.
  - New endpoints: intake_photo_upload, intake_photo_delete. Both
    HTMX-swap the gallery partial; one-at-a-time upload pattern
    (immediately POSTs on file pick — no submit button).
  - 50MB cap, image/* MIME check.
  - Photo gallery + uploader hidden behind the "I can provide photos"
    checkbox via the new .intake-reveal mechanism.

Step 3 — Website Copy
  Helpful hints under every field (About / Practice Areas / Bios)
  explaining what we expect and that polish is not required.

Step 4 — References
  Hint under reference_sites: "tell us what you like and/or don't like
  about each site — colors, layout, photos, feel. The dislikes are
  just as useful as the likes."

Step 5 — Domain & Social (renamed from "Domain & Access")
  - Domain registrar dropdown: when "Other" is selected, the JS reveals
    a required text field (`domain_registrar_other`) so we capture who
    the registrar actually is. New model field.
  - REMOVED the "I've granted Google Business Profile access" checkbox.
    GBP access is a post-launch ops task — clients don't have a reason
    to grant it before work starts. Field kept on the model for
    backward compat; just hidden from the form.
  - social_links blob split into 5 explicit URL fields:
    facebook_url, instagram_url, linkedin_url, twitter_url,
    google_business_url. The freeform `social_links` textarea now
    catches "anything else" (YouTube, TikTok, Avvo, Yelp...).

Reveal mechanism
  New intake_form.js — `.intake-reveal[data-reveal-when]` elements
  appear/hide based on a sibling input value (checkbox → checked,
  select → matching data-reveal-value). Strict-CSP-safe, no inline
  handlers. Runs once on load so server-rendered state always matches
  the live DOM.

Step labels & progress
  - "Domain & Access" → "Domain & Social".
  - Step 2 now counts as done when a photo has been uploaded (even
    without the checkbox + note).
  - Step 5 counts when domain OR any of the 5 social URLs OR the
    catch-all blob has content.

Targeted tests
  - clients: 40 passing (no regression).
